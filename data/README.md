    Аналіз прогнозних показників розвитку України (2026-2028)

Метою цього проекту є аналіз, дослідження, прогнозування та візуалізація основних прогнозних макропоказників економічного і соціального розвитку України на період 2026-2028 роки. Проєкт спрямований на аналіз ключових економічних індикаторів, схвалених Кабінетом Міністрів України, для розуміння очікуваної динаміки ВВП, ринку праці та зовнішньої торгівлі.

Датасет: Основні прогнозні показники економічного і соціального розвитку України на 2026-2028 роки

URL: https://data.gov.ua/dataset/175386f8-fbce-4352-8ec9-44fc8c436aa9/resource/3bbc64ce-4e4d-4c25-948a-be35b2abc7c0/download/osnovni-prognozni-pokazniki-ekonomichnogo-i-sotsialnogo-rozvitku-ukrayini-na-2026-2028-roki_04-0.xls

Формат датасету: xls

Набір даних містить основні прогнозні макропоказники України, такі як ВВП, індекс споживчих цін, індекс цін виробників промислової продукції, прибуток прибуткових підприємств, фонд оплати праці найманих працівників і грошового забезпечення військовослужбовців, середньомісячна заробітна плата, кількість зайнятих економічною діяльністю у віці 15–70 років, рівень безробіття, продуктивність праці, сальдо торговельного балансу, експорт та імпорт. Макропоказники розробляються на щорічній основі і схвалюються постановою Кабінету Міністрів України. За необхідності до постанови можуть вноситись зміни
Наразі не створено жодного вигляду для цього ресурсу.

Для аналізу та прогнозу будемо розглядати наступні 3 питання:

1. Чи показник імпорта переважатиме показник експорта в період 2026-2028 років?
2. Чи очікується до 2028 року зменшення рівня безробіття?
3. В якому році буде найвище значення ВВП?

Опис проєкту: Ця версія фокусується на поєднанні аналітичної логіки та високої якості вхідних даних для досягнення точних результатів.

8. Контейнеризація та Docker Workspace (Лабораторна робота №3)

У межах цієї частини проєкт було повністю переведено на мікросервісну архітектуру. 
Додаток розділено на 5 ізольованих сервісів, які оркеструються за допомогою Docker Compose, 
взаємодіють через спільні томи (Volumes) та релевантно діляться станом за допомогою СУБД SQLite.

### 1. Структура проєкту (Docker Workspace)
```text
open-data-ai-analytics/
├── data/
│   └── raw/
│       └── dataset.xls       # Вхідний макроекономічний датасет
├── src/
│   ├── data_load.py              # Сервіс імпорту даних у БД
│   ├── data_quality_analysis.py  # Сервіс аналізу якості
│   ├── data_research.py          # Сервіс розрахунку відповідей на 3 питання
│   ├── visualization.py          # Сервіс генерації графіків (matplotlib)
│   ├── web.py                    # Веб-дашборд користувача (Streamlit)
│   ├── requirements.txt          # Очищені залежності для контейнерів
│   └── Dockerfile                # Уніфікований збірник образів
├── compose.yaml                  # Маніфест мультиконтейнерної оркестрації
└── README.md
```

### 2. Опис мікросервісів додатка
data_load: Точка входу додатка. Зчитує файл dataset.xls за допомогою двигуна xlrd і розгортає реляційну базу даних SQLite, 
автоматично створюючи таблицю macro_table.
quality_analysis: Сервіс аналізу цілісності даних. Робить вибірку з SQLite, перевіряє датасет на наявність пропусків (NaN), 
дублікатів та аналізує типи даних колонок. Генерує звіт quality_report.txt.
research: Модуль аналітичних маркерів. Проводить контекстний пошук за ключовими словами для вирішення трьох дослідницьких 
питань (прогноз торгівлі, динаміка безробіття, пік ВВП). Експортує результати у структурований research_report.json.
visualization: Сервіс графічної звітності. Будує два графіки: порівняльну гістограму експорту/імпорту України та лінійний 
тренд зниження рівня безробіття (2026-2028). Зберігає зображення economic_analysis.png.
web: Серверний веб-інтерфейс на базі Streamlit. Він працює як клієнтська панель (дашборд), що зчитує згенеровані іншими 
модулями артефакти з тому й презентує графіки та результати розрахунків користувачу через браузер.

### 3. Організація обміну даними (Docker Volumes)
Взаємодія між контейнерами побудована без використання прямого мережевого обміну, а за допомогою механізмів спільного 
доступу до файлової системи та суворої оркестрації запуску:
shared_db — використовується для шерингу файлу бази даних database.db, створеного сервісом data_load. Інші аналітичні 
модулі мають до нього доступ у режимі читання.
shared_reports — виступає спільним сховищем текстових, JSON звітів та графічних фігур. Аналітичні сервіси записують туди 
артефакти, а сервіс web зчитує їх для відображення.
Щоб уникнути помилок доступу до даних (Race Conditions), у файлі конфігурації налаштовано послідовність виконання за допомогою 
умови service_completed_successfully.

### 4. Перелік портів
Сервіс web (Streamlit): внутрішній порт контейнера 8501 прокинуто на зовнішній порт хост-машини 8501. Додаток доступний 
локально за адресою: http://localhost:8501.

### 5. Інструкція та команди запуску проєкту
Повна автоматична збірка інфраструктури та старт усіх сервісів:
docker compose up --build

Зупинка роботи контейнерів та повне очищення тимчасових томів даних:
docker compose down --volumes

Перегляд статусу та стану всіх контейнерів у Docker-просторі:
docker ps -a
